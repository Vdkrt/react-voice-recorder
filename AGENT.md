# Software Design Document (SDD)
## Проект: React Audio Recorder (SPA)

**Версия:** 1.0  
**Целевые браузеры:** Safari 15+, Google Chrome (последние 2 версии), Microsoft Edge, Firefox

---

## 1. Введение

### 1.1. Назначение
Разработать одностраничное веб-приложение (Single Page Application) на базе React, позволяющее пользователю:
- Записывать звук с микрофона
- Визуально контролировать процесс записи (таймер/индикатор)
- Воспроизводить записанный аудиофайл непосредственно на странице
- Скачивать запись в формате, совместимом с браузером
- Видеть размер и длительность записи

### 1.2. Бизнес-требования
1. Запись инициируется по клику на кнопку "Записать"
2. Во время записи отображается таймер обратного отсчета или индикатор уровня сигнала
3. Остановка записи — по клику на "Стоп" или по достижении лимита времени
4. После остановки аудио автоматически становится доступным для прослушивания
5. После остановки записи, она должна прогоняться через обработку библиотекой шумоподавления @timephy/rnnoise-wasm , и сжиматься libopus-wasm.
6. На странице должны бать форма для настройки параметров библиотек libopus-wasm и @timephy/rnnoise-wasm.
7. После обработки должны быть доступны для прослушивания 3 аудиодорожки: сырая без любой обработки, после обработки шумодавом, и после сжатия через opus. Для каждой дорожки должны быть указаны размер и длительность.
5. Поддержка форматов: `audio/wav`

### 1.3. Технические ограничения
| Ограничение | Описание |
| :--- | :--- |
| **AudioContext** | Safari требует создания контекста в ответ на действие пользователя (жест) |
| **Безопасность** | Запись только по HTTPS или localhost (требование браузеров) |
| **Память** | Blob URL должны освобождаться после использования |

## 2. Архитектура приложения

### 2.1. Технологический стек
| Компонент | Технология | Назначение |
| :--- | :--- | :--- |
| **UI-библиотека** | React 18+ | Функциональные компоненты, хуки |
| **Управление состоянием** | useState, useReducer | Локальное состояние компонентов |
| **Работа с медиа** | MediaDevices API, MediaRecorder API, Web Audio API | Запись и обработка аудиопотока |
| **Обработка аудио** | libopus-wasm, @timephy/rnnoise-wasm | Шумоподавление и сжатие аудио |
| **Сборка** | Vite | Быстрая разработка и сборка |
| **Стилизация** | CSS Modules | Изолированные стили компонентов |

## 3. Структура проекта

```text
src/
  App.jsx          # главный экран приложения, логика записи и UI
  audio.js         # генерация WAV, ресемплинг, обработка аудио
  audio.test.js    # базовый тест для вспомогательной логики
  main.jsx         # точка входа React
  styles.css       # глобальные стили интерфейса
package.json       # зависимости и npm-скрипты
vite.config.js     # конфигурация Vite
```

### 3.1. Описание модулей
- App.jsx: отвечает за старт/стоп записи, работу с MediaDevices, визуальный уровень сигнала и отображение треков.
- audio.js: содержит функции для создания WAV, ресемплинга и обработки аудио через WASM-библиотеки.
- styles.css: описывает UI, карточки треков, панели настроек и состояние загрузки.
- audio.test.js: покрывает вспомогательную функцию ресемплинга.

### 3.2. Поток работы
1. Пользователь нажимает кнопку записи.
2. Приложение запрашивает доступ к микрофону и создаёт AudioContext.
3. Во время записи собираются PCM-выборки и обновляется индикатор уровня сигнала.
4. По завершении записи данные проходят обработку: шумоподавление и сжатие.
5. Генерируются три WAV-трека: raw, denoised и opus, которые доступны для прослушивания и скачивания.

### 3.3. Примечания по развитию
- Для продакшн-версии стоит добавить нормальную обработку ошибок доступа к микрофону, а также более точную интеграцию с библиотеками WASM.
- Следующий шаг — заменить упрощённый fallback-путь на более глубокую обработку в реальном аудио-потоке, если это потребуется по UX или производительности.
